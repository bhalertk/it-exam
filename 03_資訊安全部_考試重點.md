## 壹、資訊安全管理制度（ISMS）

### 一、資訊安全推動小組（第一章第一節）⭐

| 項目 | 內容 |
|------|------|
| 召集人 | 督導資訊管理群之**副總經理** |
| 副召集人 | 資訊安全部、資訊作業服務部、資訊規劃開發部**經理** |
| 委員 | 風險管理、法令遵循等單位主管及資訊單位**科長級以上** |
| 會議頻率 | 每年**一月、七月**各一次，合計**二次** |

**會議討論內容：**
先前議案追蹤、內外部議題變更、資安績效回饋、利害相關團體回饋、風險評鑑結果、持續改善建議。

**推動小組下設三分組（SR-002）⭐**
| 分組 | 職責 |
|------|------|
| **資訊安全執行分組** | 規劃執行各項資安作業、風險評鑑與處置、教育訓練 |
| **緊急處理分組** | 重大資安事件緊急應變；每**半年**辦理資訊作業持續營運計畫測試及演練 |
| **資訊安全查核分組** | ISMS內部自行查核作業 |

### 二、資訊安全風險評鑑（第一章第二節）⭐⭐

**資訊資產五大類別：**
1. **硬體**：主機/伺服器/個人電腦/機電設備
2. **軟體**：作業軟體/應用軟體/資料庫/中介軟體
3. **資料**：電子檔案（交易資料、客戶資料）
4. **文件**：書面文件（操作手冊、架構圖、申請書）
5. **人員**：內部員工及外部廠商

**風險評鑑作業：**
- 由**資訊資產保管者**每**半年**執行一次
- 結果由**資訊資產擁有者**覆核
- 現行**風險可接受水準為 17**
- 風險權值計算：**資訊資產價值 × 弱點權值 × 威脅權值**，最低1，最高**27**
- 各因子分為：高(3)、中(2)、低(1)

### 三、資訊安全指標（第一章第三節）⭐

- 量測範圍：**機密性、完整性、可用性**
- 各指標負責人員每年至少檢討**一次**
- 各項指標負責人員每月初填寫前一月份指標值
- 每**半年**於推動小組會議報告量測結果
- 按月統計之指標**連續三期**未達目標值 → 應執行矯正措施

### 四、資訊安全自行查核（第一章第四節）⭐

- 每**半年**舉行一次，範圍：整個資訊管理群
- 查核結果四等級：**完全符合 / 建議事項 / 次要缺失 / 主要缺失**
- 有不符合事項者，須於結案會議前提交「改善情形報告書」
- 改善完成後由**各部駐行稽核**覆查及用印
- 最終版報告電子檔寄至**資安規劃科**彙整

## 貳、資訊安全政策及規章（第二章）

### 一、資訊安全政策（SP-001）⭐

- 目的：確保資訊資產之**機密性、完整性、可用性及適法性**
- 適用對象：本行全體人員、**廠商、訪客**皆應遵循
- 每年**至少審查一次**
- 核定層級：**董事會**通過後實施

### 二、文件編碼規則（SR-005/第二章）⭐⭐

| 階層 | 代碼 | 全名 |
|------|------|------|
| 第一階（政策） | **SP** | Security Policy |
| 第二階（規範） | **SR** | Security Regulation |
| 第三階（作業程序） | **SW** | Security Workbook |
| 第四階（表單） | **XX-YYY-ZZZ** | 流水號自001起始 |

**紀錄管理：**
- 保存期限至少**三年**
- 銷毀需填「紀錄銷毀申請單」，經**權責主管核准**後，以**碎紙機絞碎或撕毀**等無法回復方式處理

## 肆、國外分支機構資安管理（第四章/SR-019）

### 陳報項目重點
- 海外分行資安規章至少**每年檢視一次**
- 弱點掃描結果應於**三個月內**完成風險評估及修補，完成後**一個月內**陳報總行
- 每**半年**執行一次備援演練（年度內一次同地、一次異地）
- 備援演練完成後**一個月內**陳報總行
- 資安事故應變計畫每年至少測試**一次**，完成後**一個月內**陳報

### 機房安全設備（SR-019第4條）⭐
電腦機房應配置：
1. 火災自動警報設備
2. **氣體類滅火**設備
3. 緊急用照明設備
4. 溫濕度告警裝置（含超警戒值自動通知功能）
5. **不斷電設備(UPS)**（含電力耗盡前自動通知伺服器關機功能）
6. 空調設備

### 防火牆管理重點（SR-019第8條）⭐
- 防火牆連線紀錄至少保存**五年**
- 連線規則以**正面表列**為原則，申請核准後開放，其餘一律禁止
- SWIFT電文作業使用**與外部網路隔離**之獨立電腦，不能使用電子郵件

## 伍、電子郵件安全（第五章/SR-025）⭐

### 電子郵件管理規則（SR-025第14條）⭐
- 本行**禁止使用一切外部網頁式電子郵件**
- 對內寄送郵件：系統自動過濾垃圾郵件；含**多媒體檔、執行檔、加密檔或疑似病毒**者→**隔離**
- 對外寄送郵件：含**多媒體檔、執行檔、加密檔、營業秘密或客戶資料**者→**隔離**
- 應建置**DLP機制**（Data Loss Prevention），其有效性由資安部**每年**召開會議檢視

### 危險附件副檔名（應直接刪除）
`.exe` / `.com` / `.scr` / `.bat` / `.cmd` / `.lnk`

### 社交工程演練（SR-025第16條）⭐
- 對象：**全行員工**電子郵件帳號（含海外分行）
- 頻率：每年**不定期至少二次**
- 演練郵件：每次演練期間寄發 **10封**
- 演練類型：網頁型、圖片型、巨集文件型

**測試未通過定義（以下任一）：**
1. 開啟信件並**下載圖片**
2. 開啟信件並**點選圖片連結**
3. 開啟信件並**點擊URL連結**
4. 開啟**附檔**
5. **轉寄**信件

**電子郵件放行期限：**
- 特定來源/目的之定期放行申請，期限以**半年**為限，屆期重新申請

## 柒、防毒軟體（第七章）⭐

- 病毒定義檔應保持**最新版本**
- **每月執行一次**全機掃描
- 掃描未完成比率**大於5%**之營業單位：依規予以**扣2分**
- 觸發病毒告警：需於**隔一營業日**執行ANTIVIR01報表線上簽核（決行人員：單位主管）

### USB防毒流程
1. 分行使用USB前須先透過**資安部佈署之掃毒機**執行防毒掃描
2. 確認無病毒風險才可繼續使用
3. 掃毒機發現病毒 → 將USB**還給客戶並告知含病毒**

## 捌、資訊安全事故通報（第十章/SR-004）⭐⭐

### 資安事件分類（SR-004第3條）

| 等級 | 情形 |
|------|------|
| **重大**資安事件 | ①機密等級資料遭洩漏或竄改 ②關鍵資訊作業停頓，**無法**於目標回復時間內回復 |
| **一般**資安事件 | ①內部限閱等級資料遭洩漏或竄改 ②關鍵資訊作業中斷，**可**於目標回復時間內回復 ③海外分行依當地法令通報之事件 |

### 緊急事故通報流程（第十章第一節）⭐
1. 各單位遇緊急事故 → 向**資訊作業服務部資訊服務科**通報
2. 資訊服務科於知悉或接獲通報**30分鐘內** → 通報資訊管理群群主管、資訊部門相關主管、相關業管及受影響單位
3. 若屬重大偶發事件 → 依「重大偶發事件通報作業要點」辦理
4. 若影響客戶權益 → 立即通知**客戶服務部**

### 證券期貨資通安全事件（第十章第二節）⭐
- 初步通報：知悉事件後**30分鐘內**至通報系統辦理
- 通報系統資安聯絡人：**資安管理科副科長**

### 資安事件緊急應變三階段（SW-016）⭐
| 階段 | 說明 |
|------|------|
| **事前安全防護** | 訂定預防對策、定期演練、日常網路監控、安全評估 |
| **事中主動預警緊急應變** | 蒐集危害通告、24小時監測工具偵防、即時通報應變 |
| **事後復原追蹤鑑識偵查** | 備份復原、完整紀錄建檔、向檢警申請鑑識、提報推動小組 |

**緊急應變演練（SW-016第6條）：**
- 每年**至少辦理一次**資安事件緊急應變處理演練
- 演練完成後應於**一個月內**完成演練結果報告，陳部門主管核定備查

## 玖、可辨識客戶資料管控（第十一章）⭐

### 可辨識客戶資料範圍
身分證統一編號、戶名、生日、住址、電話、**帳號、戶號、核號、信用卡卡號**、交易明細、電子郵件信箱等。

### 使用規定
- 禁止透過**社群平台、Line**等傳輸客戶個人資料
- 下載資料後**禁止複製、列印**（電子檔）；紙本**禁止影印**
- 電子檔刪除後應完成**資源回收筒清除**，由督導主管確認
- 備查簿自資料銷毀日起算至少保留**7年**

## 拾貳、雲端服務管理（SR-063）⭐

### 雲端服務三種模式
| 縮寫 | 全名 | 說明 |
|------|------|------|
| **SaaS** | Software as a Service | 使用軟體，不掌控軟硬體 |
| **PaaS** | Platform as a Service | 使用作業系統，部分掌控權 |
| **IaaS** | Infrastructure as a Service | 使用基礎運算資源，掌控OS及儲存空間 |

### 重要規定
- 雲端服務提供者須通過：**ISO 27017**（雲服務資安管理）、**ISO 27018**（雲服務個資保護）、**CSA STAR**認證
- 客戶資料及儲存地以**我國境內**為原則
- 境外儲存需求：境外資料保護法規不得低於我國；客戶重要資料應在**我國留存備份**（經主管機關核准者除外）
- 規章每年至少**檢視一次**
- 具重大性境外雲端委外服務：每年至少辦理**一次查核**
