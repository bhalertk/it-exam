## 參、資訊專案管理（第四章）⭐

### 審議依據規章
- 「華南商業銀行**採購電腦軟硬體相關設備**注意事項」
- 「華南商業銀行電腦軟硬體相關設備**審議小組**設置暨作業注意事項」
- 「華南商業銀行**資訊系統概念驗證作業**須知」

### 專案採購流程
1. 審議小組審核通過
2. 資訊部門擬具簽請採購書類，檢附規格文件
3. 會簽**行政管理部、財務會計部**與相關部門
4. 報請權責層級核定
5. **行政管理部**依採購相關規定辦理採購事宜

### 專案執行管理
- 專案會議紀錄應至少每**二個月**彙整一次，簽報單位主管核定
- 遇**重大議題**（如架構調整）→應**立即**簽報
- 專案無法如期上線→期限屆滿前邀集業管單位評估

## 肆、電腦作業應用程式管理（SR-010）⭐⭐⭐

### 應用程式分類

| 類型 | 帳務類 | 非帳務類 |
|------|--------|---------|
| **連線作業** | 連線帳務類（與會計帳務/客戶入扣帳有關） | 連線非帳務類 |
| **整批作業** | 批次帳務類（選檔、異動程式） | 批次非帳務類 |

### 系統開發五階段⭐

```
可行性分析 → 需求分析 → 系統分析 → 程式撰寫與測試 → 驗收上線
```

### 可行性分析報告內容（第5條）
1. 定義專案範圍、**成本與效益**及必要性
2. 專案時程
3. 使用技術、系統架構
4. **自行開發或委外**開發之建議
5. 本行預估投入人力
6. 綜合評估結果

> 專案時程小於**二百人日**者，得逕予辦理（免可行性分析報告）

### 程式撰寫規定⭐
- 各項專案應用程式應分別交付**二人以上**程式員撰寫
- 撰寫應在**測試**程式資料集或目錄內
- 開發中程式應**定期備份**
- 程式測試時，限使用**測試環境**

### 需要CODE REVIEW的程式（第29條）⭐⭐
1. 使用者登錄及**權限與密碼**有關之程式
2. **海外網路銀行轉帳**交易
3. **連線帳務類**程式
4. **批次帳務類**程式
5. 報送**聯徵中心授信資料**之程式

### APP管理（第30、31條）⭐
- APP首次發布或**權限變動**→需經**資規部單位法遵主管、資安部、法遵部、風管部**審視
- APP委外開發且未取得原始碼者→須使用本行**防毒軟體掃描**，並要求廠商提供**原始碼檢測或黑箱檢測報告**

### 驗收文件（第18條）⭐
1. 需求分析報告書
2. 系統分析規格書
3. 系統設計規格書
4. 程式設計規格書
5. 測試計畫書
6. 測試個案說明書
7. 測試報告書
8. 上線計畫書
9. 原始程式
10. 操作手冊
11. 其他文件

> 文件得以儲存媒體取代，須製成**兩份**供**同異地存放**

### 程式報廢計畫（第37條）
報廢計畫內容：報廢目的、對象範圍、日期、方法、會計帳資產之處分方法

### 核定層級
**總經理**核定後實施

## 伍、資訊作業變更管理（SR-013）⭐⭐⭐

### 變更範圍七大類
1. 系統硬體
2. 系統軟體
3. 資訊網路
4. 資料檔(庫)之結構或種類改變
5. 資料檔(庫)內容更改
6. 應用程式
7. 系統資源使用及存取權限變更

### 變更管理程序（五步驟）⭐
1. **申請及初核**
2. **技術及業務審核**（重要變更→召開變更管理會議）
3. **測試及驗證**
4. **排程及同意變更**
5. **變更實施及覆核結案**

### 核可層級⭐
| 等級 | 核可 |
|------|------|
| 一般變更 | 申請單位**單位副主管**以上 |
| 重要變更 | 申請單位**單位主管** |

### 變更管理會議（第14條）⭐⭐
- 涉及**第12條第1項第6款**重要變更→由**資訊安全長**主持
- 其餘由**資訊作業服務部部主管**主持
- 成員：資訊管理群轄下部門部(副)主管、各科主管、變更管理員
- 變更管理員由**資訊內控管理科**科主管指派

### 上線協調會議（第15~16條）
- 涉及第12條第6款→由**資訊安全長**主持
- 其餘由**重要變更案件申請單位之部主管**主持
- 成員含：客戶服務部、法令遵循部、行政管理部公關科

### 農曆春節變更管制（第29條）⭐
- **假期始日前一個月**起至假期結束，原則禁止營運環境變更
- 例外須經**業管單位群群主管及資訊管理群群主管**書面同意

### 核定層級
**總經理**核定後實施

## 陸、使用者權限管理

### 系統使用者權限管理要點⭐⭐

**權限管理原則（第5條）：**
1. **最小權限**：帳號權限授予以履行工作職責所需為原則
2. **職務分工與牽制**：避免單一使用者掌握過高權限
3. 帳號申請使用/變更/刪除，應先取得**單位主管同意**，並保留稽核軌跡

**權限審查頻率（第6條）：**
| 系統類別 | 清查頻率 |
|---------|---------|
| 資訊系統 | 至少**每年一次** |
| 業務應用系統 | 至少**每半年一次** |

- 資訊規劃開發部：每**半年**清查及更新**系統清單**
- 核定層級：經**董事長**核定後實施

### 電腦作業應用系統使用者授權（SW-004）⭐

**雙重管控機制：**
- 新增或修改使用者授權時，主管設定後須由**另一位主管進行覆核**（雙重管控原則）
- 核定層級：**資訊規劃開發部部主管**

## 柒、外包人員管理（SR-024）⭐

### 門禁管制重點
1. 進駐前須提交：**外包室使用申請表、進駐人員名冊、外包室使用同意書**
2. 進入時：保全服務台確認、繳交**身分證**換取**出入證**、簽到
3. 出入證應設定**電梯樓層管制**，僅得進出**外包室及餐廳樓層**
4. 假日或下班時段需**同時於兩簿登記**（訪客登記簿+廠商簽到簿）
5. 進出**機房或伺服器作業連線室**→應由本行人員**陪同**

### 設備場所使用規定
- 外包室網路為**獨立網段**，以防火牆與內部網路區隔
- 僅可申請連接**測試環境**網段，**不得**連結本行其他內部網路或**網際網路**
- 確需使用或變動本行場所/軟硬體設定→由本行**外包專案負責人**提出申請，經**主管核准**
- 限使用指定場所之機器；非與本行人員開會**不可佔用會議室**
- 下班時應**關機**，將門窗、水電、空調**關妥**
- 未經許可不得**拆解、移除、增添**本行設備
- 未經許可不得將本行檔案、程式及文件以任何方式**複製、傳輸及攜出**
- 未經許可不得使用本行**影印、傳真**及其他辦公室設備

### 禁止行為（第5條第三款）⭐
1. 使用**P2P軟體、即時通訊**、登入不當網站、收發Email或視訊會議
2. **下載或上傳**任何檔案
3. **大量上網**或影響流量
4. 安裝**側錄、監控、掃描或遠端遙控**軟體
5. 對本行設備做**探索或蒐集**網路資料

## 玖、程式撰寫安全規則（SW-007）⭐⭐

### 密碼安全（第7、8條）⭐
- **密碼不得以明碼形式存放於資料庫或檔案**中
- **不得將使用者代碼及密碼寫入程式模組**（程式中禁止 hard-code 帳號密碼）

### 程式頭部註解（第13條）
每支程式**開頭須有註解區段**，應包含以下6項：
1. 程式名稱
2. 功能簡述
3. 撰寫日期
4. 修改日期
5. 修改內容
6. 修改人名稱

### 測試資料管理（第17條）⭐⭐
- **正式環境資料不得複製至測試環境或驗證環境**使用
- 如必須複製，**可辨識客戶資料**須依「可辨識客戶資料去識別化原則」進行**遮蔽或亂碼處理**
  - 例外：配合外部機構測試演練、新專案開發或系統轉換之系統驗證者，不在此限
- 應召開會議建立管控機制（複製範圍、作業方式、存取權限），會議紀錄須簽報**部主管**核定
- 未去識別化之可辨識客戶資料，存取權限須**比照正式環境**，且資料使用完畢後**須執行刪除**
- 相關簽核紀錄及表單**至少留存五年**
- **資料不得存放於個人電腦**

## 拾、品質管理（SR-030）⭐

### 測試等級

| 等級 | 適用範圍 | 測試依據 |
|------|---------|---------|
| **第一級** | 需求變動造成程式功能改變、新增程式、瑕疵改善、畫面變動、整批程式異動 | 依**程式設計規格書**編製測試個案 |

1. 程式變更清單
2. 程式變更前後比對報表
3. 程式測試報告書
4. 程式設計規格書
5. 程式碼檢核表
6. 需求單或其他變更依據文件
7. 批次作業之 JCL

### 測試產出文件（4項）
1. 品管測試計劃
2. 品管測試個案紀錄
3. 品管測試**異常報告書**

1. 相關測試個案**全部測試完成**
2. 異常項目均已完成更正與**覆測**（覆測範圍須包含所有與修正部分相關之個案）
3. 品管測試報告書撰寫完成

**總經理**核定後實施

## 拾壹、應用程式原始碼管理（SW-011）⭐⭐

### 環境隔離原則⭐
- 開發環境應與正式環境**分別獨立於不同實體主機**
- 若無法分別使用不同實體主機，至少應**邏輯隔離**

### 原始碼保存規定⭐

| 系統類型 | 原始碼保存代數 | 執行檔版次 | 備份頻率 | 保留期間 |
|---------|-------------|----------|---------|---------|
| **帳務主機** | 至少保留 **7代** | 至少保留 **3代** | **每週**備份 | 保留**半年** |
| **開放系統** | 至少保留 **7代** | 至少保留 **7代** | **每日**備份 | 保留**一週** |

### 其他規定
- 版控系統應定期執行程式原始碼**異地備份**作業
- 應評估**避免**於正式營運環境安裝程式原始碼

### 核定層級
**資訊規劃開發部經理**核定後實施

## 拾貳、API開發管理（SW-034）⭐⭐

### API風險分類⭐

| 風險等級 | 說明 |
|---------|------|
| **低風險** | 查詢公開資料 |
| **中風險** | 涉及金融帳戶或產品資料 |
| **高風險** | 涉及交易指示 |

### API安全要求⭐
- 機敏資料於**網際網路傳輸**應全程**加密**
- 固定密碼不應於網際網路上以**明碼**傳輸
- **認證資訊不得以URL參數形式傳送**（應使用 Authorization Header 或 Body）
- **API Key 使用期限不得超過5年**
- 提供予 **TSP業者** 的API，需依 **OWASP Top 10 API Security Risks** 辦理

### 定期盤點
- 資規部每**半年**盤點對外服務存取之 API

### 核定層級
**資訊規劃開發部部主管**核定後實施

## 拾伍、高頻考點速記

```
【系統開發流程】
可行性分析 → 需求分析 → 系統分析 → 撰寫測試 → 驗收上線
- 小於200人日者→得逕予辦理（免可行性分析報告）
- 程式撰寫：二人以上、測試環境、定期備份

【CODE REVIEW五種程式】
1.登錄/權限/密碼 2.海外網銀轉帳 3.連線帳務 4.批次帳務 5.聯徵授信

【變更管理五步驟】
申請初核 → 技術業務審核 → 測試驗證 → 排程同意 → 實施覆核結案

【變更核可層級】
- 一般：副主管以上 / 重要：單位主管
- 第12條第6款重要變更→資訊安全長主持

【農曆春節】假期前一個月起禁止營運環境變更

【外包人員】
- 獨立網段/僅測試環境/禁止連結內部網路及網際網路
- 禁止P2P、即時通訊、上傳下載
- 進出機房須本行人員陪同

【使用者權限管理】
- 最小權限 / 職務分工與牽制
- 資訊系統清查：每年一次 / 業務應用系統：每半年一次
- 核定：董事長

【SW-004 使用者授權雙重管控】
新增/修改授權 → 主管設定後，另一位主管覆核
核定：資規部部主管

【APP發布】
- 權限變動→法遵主管+資安部+法遵部+風管部審視
- 委外APP→防毒掃描+原始碼或黑箱檢測報告

【驗收文件】兩份，同異地存放

【SW-007 程式安全規則】
- 密碼禁止明碼存DB/檔案
- 帳密禁止寫入程式碼（hard-code）
- 程式頭部必須有6項註解（名稱、功能、撰寫日期、修改日期、修改內容、修改人）
```
